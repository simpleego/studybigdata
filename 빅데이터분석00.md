## **미리 보기** [cite: 19]

### **접근 전략** [cite: 20]

빅데이터 분석 기획은 빅데이터를 분석하기 위한 가장 기초가 되는 단원입니다. [cite: 22] 빅데이터의 유형, 가치, 산업, 조직 구조 등을 다양한 관점에서 볼 수 있게 합니다. [cite: 23] 또한, 가장 수월하게 점수를 획득할 수 있는 단원에 해당하여 기본개념들을 잘 알아 두시면 합격점수 획득에 큰 도움이 되겠습니다\! [cite: 24]

### **미리 알아두기** [cite: 21]

  * **빅데이터**: 막대한 양의 정형 및 비정형 데이터이며, 데이터로부터 가치를 추출하고 결과를 분석하는 기술의 의미로 통용됩니다. [cite: 25, 26]
  * **NoSQL**: 데이터 저장에 고정된 테이블 스키마가 필요하지 않고 조인(Join) 연산을 사용할 수 없으며, 수평적으로 확장이 가능한 DBMS입니다. [cite: 27, 28]
  * **하둡 분산 파일 시스템(HDFS)**: 수십 테라바이트 또는 페타바이트 이상의 대용량 파일을 분산된 서버에 저장하고, 그 저장된 데이터를 빠르게 처리할 수 있게 하는 파일 시스템입니다. [cite: 29, 30]
  * **빅데이터 분석 방법론**: 빅데이터를 분석하기 위해 문제를 정의하고 답을 도출하기 위한 체계적인 절차와 처리 방법입니다. [cite: 31, 32]
  * **빅데이터 플랫폼**: 빅데이터에서 가치를 추출하기 위해 일련의 과정(수집 저장 처리 분석 시각화)을 규격화한 기술입니다. [cite: 33, 34]
  * **데이터 비식별화**: 데이터를 안전하게 활용하기 위하여 수집된 데이터의 개인정보 일부 또는 전부를 삭제하거나 다른 정보로 대체함으로써 다른 정보와 결합하여도 특정 개인을 식별하기 어렵게 하는 기술입니다. [cite: 35, 36]

### **핵심 키워드 베스트 일레븐(Best Eleven)** [cite: 37, 38]

빅데이터, 정형 데이터, 비정형 데이터, HDFS, NoSQL, ETL, JSON, 데이터 변환, 데이터 비식별화, 개인정보보호, 빅데이터 보안 [cite: 39]

## **Chapter 01: 빅데이터의 이해** [cite: 50]

### **1. 빅데이터 개요 및 활용** [cite: 52]

#### **(1) 빅데이터 특징** [cite: 57]

**① 빅데이터(Big Data) 개념** [cite: 59]

  * 빅데이터는 막대한 양(수십 테라바이트 이상)의 정형 및 비정형 데이터입니다. [cite: 60]
  * 데이터로부터 가치를 추출하고 결과를 분석하여 통찰, 지혜를 얻는 과정입니다. [cite: 61]
  * Ackoff, R.L이 도식화한 DIKW 피라미드로 표현할 수 있습니다. [cite: 62]

**DIKW 피라미드** [cite: 63]
 [cite: 74, 75, 76, 77, 78, 79, 80, 81, 82]

| 피라미드 요소 | 설명 |
| :--- | :--- |
| **데이터 (Data)** | 객관적 사실로서 다른 데이터와의 상관관계가 없는 가공하기 전의 순수한 수치나 기호. [cite: 64] 예: 수제비 빅분기 책을 A 사이트에서 30,000원, B 사이트에서 35,000원에 판매. [cite: 64] |
| **정보 (Information)** | 가공, 처리하여 데이터 간의 연관 관계와 함께 의미가 도출된 요소. [cite: 64] 예: 수제비 빅분기, 정처기 책은 A 사이트에서 더 싸게 판매. [cite: 64] |
| **지식 (Knowledge)** | 획득된 다양한 정보를 구조화하여 유의미한 정보로 분류하고 일반화시킨 결과물. [cite: 64] 예: 서적들은 A 사이트가 싸게 팔기 때문에 수제비 책을 구입할 계획. [cite: 64] |
| **지혜 (Wisdom)** | 근본 원리에 대한 깊은 이해를 바탕으로 도출되는 창의적 아이디어. [cite: 64] 예: A 사이트의 다른 상품들도 B 사이트보다 저렴할 것으로 판단. [cite: 64] |

  * 기존의 관리 방법으로는 처리하기 어려운 막대한 양의 데이터를 처리할 때 빅데이터를 사용합니다. [cite: 83, 84]

**② 데이터의 양을 측정하는 단위** [cite: 85]

| 기호 | 이름 | 값 |
| :--- | :--- | :--- |

| MB | 메가바이트 | $1MB=10^{3}KB=10^{6}Bytes$ [cite: 87] |
| GB | 기가바이트 | $1GB=10^{3}MB=10^{9}Bytes$ [cite: 87] |
| TB | 테라바이트 | $1TB=10^{3}GB=10^{12}Bytes$ [cite: 87] |
| PB | 페타바이트 | $1PB=10^{3}TB=10^{15}Bytes$ [cite: 87] |
| EB | 엑사바이트 | $1EB=10^{3}PB=10^{18}Bytes$ [cite: 87] |
| ZB | 제타바이트 | $1ZB=10^{3}EB=10^{21}Bytes$ [cite: 87] |
| YB | 요타바이트 | $1YB=10^{3}ZB=10^{24}Bytes$ [cite: 87] |

#### **(2) 빅데이터 특성** [cite: 94]

빅데이터는 전통적으로 3V (Volume, Variety, Velocity)의 특성이 있으며, 최근에는 4V (Value 추가), 5V (Veracity 추가), 7V (Validity, Volatility 추가)로 확장되고 있습니다. [cite: 95]

| 특성 | 설명 |
| :--- | :--- |
| **규모 (Volume)** | 빅데이터 분석 규모와 관련된 특징으로, ICT 기술 발전으로 디지털 정보량이 기하급수적으로 증가하는 것을 의미합니다. [cite: 99] |
| **다양성 (Variety)** | 빅데이터 자원 유형에 관련된 특징으로, 정형 데이터뿐만 아니라 비정형, 반정형 데이터를 포함합니다. [cite: 109] |
| **속도 (Velocity)** | 빅데이터 수집·분석·활용 속도에 관련된 특징으로, 실시간성 정보의 생성 속도 증가에 따라 처리 속도 가속화가 요구됩니다. [cite: 109] |
| **가치 (Value)** | 빅데이터 수집 데이터를 통해 얻을 수 있는 가치로, 비즈니스나 연구에 활용되어 유용한 가치를 끌어낼 수 있는지를 의미합니다. [cite: 109] |
| **신뢰성 (Veracity)** | 빅데이터 수집 대상 데이터가 가지는 신뢰에 관련된 특징으로, 노이즈 및 오류 제거를 통해 데이터 품질과 신뢰성을 높여야 합니다. [cite: 109] |
| **정확성 (Validity)** | 빅데이터 수집 대상 데이터가 가지는 유효성과 정확성으로, 질 높은 데이터를 활용한 정확한 분석이 중요합니다. [cite: 109] |
| **휘발성 (Volatility)** | 빅데이터 수집 대상 데이터가 의미를 가지는 기간으로, 데이터가 얼마나 오래 저장되고 타당하게 쓰일 수 있는지에 관한 사항입니다. [cite: 109] |

#### **(3) 데이터 지식경영** [cite: 110]

  * 빅데이터는 장기적인 관점에서 유용한 가치를 창출해야 합니다. [cite: 113]
  * 데이터 기반 지식경영의 핵심은 암묵지와 형식지의 상호작용에 있습니다. [cite: 114]

**지식 구분** [cite: 115]

| 구분 | 설명 |
| :--- | :--- |
| **암묵지 (Tacit Knowledge)** | 학습과 경험을 통해 개인에게 체화되어 있지만 겉으로 드러나지 않는 지식입니다. [cite: 116] 예: 수영, 태권도. [cite: 116] |
| **형식지 (Explicit Knowledge)** | 문서나 매뉴얼처럼 형상화된 지식으로 전달과 공유가 용이합니다. [cite: 116] 예: 수험서, 소프트웨어 설치 매뉴얼. [cite: 116] |

**데이터 지식경영 상호작용 (SECI 모델)** [cite: 124, 125]
 [cite: 117, 118, 119, 120, 121, 122, 123]

| 상호작용 | 내용 |
| :--- | :--- |
| **공통화 (Socialization)** | 다른 사람과의 상호작용을 통해 개인이 암묵지를 습득하는 단계. [cite: 127] |
| **표출화 (Externalization)** | 개인에게 내재된 경험을 객관적인 데이터(문서, 매체)로 저장하거나 가공, 분석하는 과정. [cite: 127] |
| **연결화 (Combination)** | 형식지가 상호 결합하여 새로운 형식지를 창출하는 과정. [cite: 127] |
| **내면화 (Internalization)** | 행동과 실천교육 등을 통해 형식지가 개인의 암묵지로 체화되는 단계. [cite: 127] |

## **Chapter 02: 데이터 분석 계획** [cite: 1492]

### **1. 분석 방안 수립** [cite: 1496]

#### **(1) 하향식 접근 방식 (Top Down Approach)** [cite: 1537]

  * **개념**: 분석 과제가 이미 정해져 있고, 이에 대한 해법을 찾기 위해 체계적으로 분석하는 방법입니다. [cite: 1539]
  * **절차**: 문제 탐색 → 문제 정의 → 해결방안 탐색 → 타당성 검토 → 선택의 과정을 거칩니다. [cite: 1542]

| 단계 | 설명 |
| :--- | :--- |
| **1. 문제 탐색 (Problem Discovery)** [cite: 1555] | 비즈니스 모델을 기반으로 문제를 탐색하고 분석 기회를 발굴합니다. [cite: 1556] |
| **2. 문제 정의 (Problem Definition)** [cite: 1561] | 비즈니스 문제를 데이터 문제로 변환하여 정의하고, 필요한 데이터 및 기법을 정의합니다. [cite: 1567, 1568] |
| **3. 해결방안 탐색 (Solution Search)** [cite: 1576] | 정의된 문제를 해결하기 위한 다양한 분석 방안을 탐색합니다. [cite: 1569] |
| **4. 타당성 검토 (Feasibility Study)** [cite: 1584] | 경제적, 데이터 및 기술적 타당성을 평가합니다. [cite: 1590, 1591] |
| **5. 선택 (Selection)** [cite: 1588] | 타당성에 입각하여 최적의 대안을 선택하고 프로젝트화합니다. [cite: 1593] |

#### **(1) 데이터 수집** [cite: 2550]

수집 데이터는 위치에 따라 **내부 데이터**와 **외부 데이터**로 구분됩니다. [cite: 2552]

#### **(2) 데이터 수집 방식 및 기술** [cite: 2563]

  * **ETL (Extract Transform Load)**: 다양한 소스로부터 데이터를 **추출(Extract)**, \*\*변환(Transform)\*\*하여 데이터 웨어하우스(DW) 등에 \*\*적재(Load)\*\*하는 기술입니다. [cite: 2568]
  * **스쿱 (Sqoop)**: 관계형 데이터베이스(RDBMS)와 하둡 파일 시스템(HDFS) 간에 데이터를 수집하고 전송하는 기술입니다. [cite: 2625]
  * **플럼 (Flume)**: 대량의 로그 데이터를 효율적으로 수집, 집계, 이동하기 위해 이벤트와 에이전트를 활용하는 기술입니다. [cite: 2719, 2720]
  * **크롤링 (Crawling)**: 웹 사이트로부터 웹 문서 및 콘텐츠를 수집하는 기술입니다. [cite: 2842]
  * **Open API**: 공개된 API를 통해 실시간으로 데이터를 수신하는 기술입니다. [cite: 2846]

| | **비정형 데이터** [cite: 2899] | 정해진 구조가 없는 데이터 [cite: 2899] | 텍스트, 이미지, 오디오, 비디오, SNS 데이터 [cite: 2899] |

| | **스트림 데이터** [cite: 2928] | 네트워크를 통해 실시간으로 전송되는 데이터 [cite: 2928] | 센서 데이터, HTTP 트랜잭션 [cite: 2928] |

### **3. 데이터 비식별화** [cite: 2937]

  * **개념**: 특정 개인을 식별할 수 없도록 개인정보의 일부 또는 전부를 변환하는 방법입니다. [cite: 2940]
  * **처리 기법**: 가명처리, 총계처리, 데이터 값 삭제, 범주화, 데이터 마스킹 등이 있습니다. [cite: 2968]

| 처리 기법 | 설명 | 예시 |
| :--- | :--- | :--- |
| **가명처리** [cite: 2972] | 식별 가능한 데이터를 다른 값으로 대체하는 기법 [cite: 2972] | '홍길동' → '임꺽정' [cite: 2972] |
| **총계처리** [cite: 2974] | 데이터를 통계값으로 적용하여 특정 개인을 판단할 수 없게 하는 기법 [cite: 2974] | 'A: 170cm, B: 180cm' → '평균 키: 175cm' [cite: 2974] |
| **데이터 값 삭제** [cite: 2974] | 개인 식별이 가능한 특정 데이터 값을 삭제하는 기법 [cite: 2974] | '주민등록번호 901212-1234567' → '90년대생, 남자' [cite: 2974] |
| **범주화** [cite: 2975] | 단일 식별 정보를 대푯값이나 구간 값으로 변환하는 기법 [cite: 2975] | '41세' → '40대' 또는 '40\~50세' [cite: 2975] |
| **데이터 마스킹** [cite: 2977] | 개인 식별 정보의 전체 또는 일부를 대체 값(\*, 공백 등)으로 변환하는 기법 [cite: 2977] | '홍길동' → '홍\*\*' [cite: 2977] |
